More Configuration

Currently, the configuration options available for the input and output plugins (the output especially) are rather paltry. Several items need to be configurable:

  • Should be able to exclude/include specific schemas in input plugins
  • Should be able to disable certain output items (e.g. diagrams, highlighted SQL, etc.)
  • Should be able to generate absolute URLs instead of relative in HTML output (which will require some knowledge of the final URL under which the documentation output will be accessible)
  • Should be able to specify diagram file encoding (including inline or included SVG instead of PNG+image-map)
  • Should be able to specify related links to include in all documents

Refinement of Output

Several items in the output need some extra refinement. Specifically, graphs should never be ridiculously large. They should vary the amount of information they display to accomodate a reasonably sized display (e.g. 1024x768). A glossary or index list of all documented objects would also be useful along with expandable lists of objects in each document to allow near-universal navigation from anywhere.

Support Other Databases

Add new input plugins for other databases. Probably in the following order:

  1. IBM DB2 UDB 8+ for z/OS
  2. PostgreSQL 8+
  3. Oracle 9 or 10
  4. Interbase/Firebird
  5. MySQL 5+

The reason for the ordering is for similarity of the target dialect to that of DB2 UDB for Linux/UNIX/Windows, and for the availability of meta-data in the target platform (hence why it's likely that only MySQL 5 or above will ever be supported - 4 and below have such a weird and non-standard dialect that the effort required is simply too great).

Support Other Output Formats

I would like to add output plugins for the following formats, probably in this order:

  1. HTML Javadoc-style output
  2. Docbook style output (could be used for PDF and other formats suited to printing)
  3. Kid templates (for producing output in an arbitrary XML dialect)

Work on the last has already started but probably won't be completed until the first two have been completed.
